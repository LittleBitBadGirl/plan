## 📸 Импорт созвонов из Яндекс.Календаря

**Workflow:**
1.  Пользователь делает скриншот недели в Яндекс.Календаре.
2.  Отправляет скриншот в чат.
3.  LLM распознает встречи и формирует команду для терминала.
4.  Пользователь копирует команду и запускает в терминале.

**Скрипт:** `import_meetings.py`
-   Принимает JSON-строку со списком встреч.
-   Ищет категорию "🏢 Работа" и подкатегорию "Созвоны".
-   Добавляет задачи, проверяя на дубликаты.

**Пример команды:**
```bash
cd /Users/vera/Desktop/личные_доки/"СLI"/plan
source .venv/bin/activate
python import_meetings.py '[{"title": "1-1 Лена", "date": "2026-04-16"}]'
```

## 💡 Важные решения

1. **Аутентификация отключена** — локальный комп, токен не нужен
2. **Qwen в fallback-режиме** — нет HF_TOKEN, категоризация по ключевым словам
3. **Категории:** Работа (Финансы, Документы, Проекты, Инфраструктура, Контент, Созвоны), Личное (Семья, Здоровье, Финансы, Цифровая гигиена, Покупки, Социум, Пет-проекты, Свои сайты), Обучение (Курсы, Инструменты, Контент), Периодические (Встречи, Забота о себе)
4. **Бэклог** — задачи без даты, откуда планировать на день (⚡), на дату (📅) или превращать в периодические (🔄)
5. **Хронические задачи** — перенесённые >7 раз, флаг `chronic_task=True`
6. **Telegram бот → n8n** — polling на сервере 24/7, сообщения не теряются
7. **Пагинация архива** — 50 задач на страницу
8. **Порт 8000** — зарезервирован под планировщик (см. ~/.qwen/QWEN.md)
9. **Умный перенос (Rollover):**
    - В рабочие дни: переносит всё.
    - В выходные (Сб/Вс): переносит только личные задачи. Рабочие задачи ждут до понедельника (без увеличения счетчика переносов).
10. **Бэклог → Периодические** — кнопка 🔄 превращает задачу в шаблон и удаляет из бэклога
11. **Бэклог → На сегодня** — кнопка ⚡ мгновенно ставит дату сегодня
12. **Подзадачи (Subtasks):**
    - **Скрыты по умолчанию.** Кнопка 📂 в строке с категорией открывает список.
    - **Визуально компактно:** `Категория | 📂 1/3 | ⏱ x1`.
    - **Форма добавления:** Появляется внутри раскрытого списка.
    - **Каскадное закрытие:** Закрытие всех детей → закрытие родителя.
13. **Восстановление из архива:**
    - Кнопка "↩️ Вернуть" возвращает задачу из архива в активный список.
    - Статус сбрасывается на "новая", `completed_at` обнуляется.
14. **Статистика:**
    - Фильтр по корневым задачам (без подзадач), чтобы не дублировать счетчики.
    - Метрики: Активные, Выполненные, Новые, Без категории, Средняя скорость выполнения.
15. **Архитектура интеграции:** n8n (BY) → Cloudflare Tunnel → Localhost:8000
