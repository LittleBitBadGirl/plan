## 1. Обзор проекта

Персональный планировщик задач с Telegram-ботом и веб-интерфейсом, AI-категоризацией через Qwen, распознаванием скриншотов календаря и AI-анализом нагрузки.

### 1.1. Ключевые функции

- ✅ Создание задач через Telegram или веб-интерфейс
- ✅ AI-категоризация задач (глобальные категории → подкатегории)
- ✅ Распознавание скриншотов Яндекс.Календаря
- ✅ Управление подзадачами
- ✅ Периодические задачи с расписанием
- ✅ AI-анализ нагрузки и рекомендации
- ✅ Архив задач
- ✅ Просмотр задач на любую дату
- ✅ Синхронизация пропущенных из Telegram при включении

### 1.2. Пример структуры категорий

```
🏢 Работа
├── Финансы (маржа, аналитика, подсчёт денег)
├── Документы (реестры, инлайн-документация)
├── Проекты (зелёный марафон, майоли, создание ТЗ для клиентов)
├── Инфраструктура (серверы, деплой, VPN для работы)
└── Контент (статьи, выступления для работы)

🏠 Личное
├── Семья (Даня, подарки)
├── Здоровье (врач, массаж, баня)
├── Финансы (оплата света, личные финансы)
├── Цифровая гигиена (битварден, фото, скрины)
├── Покупки
├── Социум (друзья)
├── Пет-проекты (планербот, транскрибатор)
└── Свои сайты (ТЗ свой сайт, сайт для Дони садоводства)

📚 Обучение
├── Курсы (SQL, продакт)
├── Инструменты (github, C4 диаграммы)
└── Контент (Замесин, обучающие видео)

🔄 Периодические
├── Встречи с друзьями (пт)
└── Забота о себе (врач/массаж/баня)
```

**Примечание:** Категории настраиваются пользователем через веб-интерфейс и Telegram бота.

## 3. Поток данных

### 3.1. Добавление задачи

1. **Источник:** Telegram (текст/голос/скриншот) или Веб (форма)
2. **Голос → текст:** GigaAM транскрибирует → текст задачи
3. **Скриншот → события:** OCR распознаёт → извлечение времени и названий → создание задач
4. **AI категоризация:** Qwen анализирует текст → определяет категорию и подкатегорию
5. **Сохранение:** Задача в БД со статусом "новая"
6. **Уведомление:** 
   - Telegram: "✅ Задача добавлена: [название] [Категория/Подкатегория]"
   - Веб: задача появляется на дашборде

### 3.2. Синхронизация пропущенных

1. При запуске бэкенд проверяет таблицу `missed_messages`
2. Обрабатывает все необработанные сообщения (атомарное обновление флага `processed`)
3. Создаёт задачи из текста/голосовых/скриншотов
4. Отправляет сводку: "Обработано X пропущенных сообщений, создано Y задач"

### 3.3. Асинхронная AI-обработка

```
1. Пользователь создаёт задачу → мгновенный ответ "✅ Задача создана"
2. Задача попадает в asyncio.Queue
3. Воркер берёт задачу из очереди:
   ├─ Вызывает Qwen для категоризации (5-15 сек)
   ├─ При успехе → обновляет category_id
   ├─ При ошибке → ставит категорию "Без категории" + флаг needs_review
   └─ Отправляет уведомление: "📂 Задача категоризирована: Работа/Созвоны"
4. Ошибка AI не блокирует создание задачи
```

### 3.4. Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Qwen недоступен/ошибка | Категория "Без категории", флаг needs_review, повтор через 5 мин |
| OCR не распознал текст | Сохранить скриншот, создать задачу "Распознать вручную", ссылка на изображение |
| GigaAM ошибка транскрибации | Сохранить голосовое, создать задачу "Транскрибировать вручную" |
| SQLite заблокирован | Retry с экспоненциальной задержкой (1с, 2с, 4с) |
| Дубликат сообщения | Проверка по hash(message_text + created_at) |

### 3.5. Перенос невыполненных задач

```
1. Ежедневно в 00:05 APScheduler проверяет:
   ├─ Задачи со статусом "новая" и due_date < сегодня
   ├─ Задачи со статусом "в_работе" и due_date < сегодня
   
2. Для каждой просроченной задачи:
   ├─ Увеличить счётчик postpones (переносов)
   ├─ Обновить due_date = сегодня
   ├─ Если postpones > 7 → флаг chronic_task
   └─ Отправить уведомление: "⚠️ Задача переносится 8-й раз"

3. AI-анализ хронических задач (раз в неделю):
   ├─ Собрать все chronic_task задачи
   ├─ Проанализировать паттерны (похожие категории, темы, сложность)
   ├─ Сгенерировать рекомендации:
   │   "5 из 8 хронических задач в категории 'Личное/Документы'.
   │    Возможно, стоит делегировать или упростить процесс."
   └─ Сохранить отчёт в /stats
```

## 7. Фоновый планировщик (APScheduler)

### 7.1. Задачи планировщика

| Задача | Расписание | Описание |
|--------|------------|----------|
| generate_recurring_tasks | daily 00:05 | Создать задачи из шаблонов на сегодня |
| rollover_overdue_tasks | daily 00:10 | Перенести просроченные задачи |
| cleanup_missed_messages | daily 03:00 | Удалить обработанные > 30 дней |
| compute_daily_stats | daily 23:55 | Агрегировать статистику за день |
| analyze_chronic_tasks | weekly mon 09:00 | AI-анализ хронических задач |
| update_context_from_feedback | weekly sun 20:00 | AI анализирует feedback и обновляет контекст |
| cleanup_old_screenshots | weekly | Удалить скриншоты > 90 дней |
| retry_failed_ai | hourly | Повторить категоризацию failed задач |

### 7.2. Генерация периодических задач

```python
# Псевдокод
for template in recurring_tasks(is_active=True):
    if template.recurrence_type == 'daily':
        create_task(title=template.title, due_date=today, time=template.time_of_day)
    elif template.recurrence_type == 'weekly':
        if today.weekday() in template.recurrence_days:
            create_task(title=template.title, due_date=today, time=template.time_of_day)
    elif template.recurrence_type == 'monthly':
        if today.day == template.start_date.day:
            create_task(title=template.title, due_date=today, time=template.time_of_day)
```

### 7.3. Обновление контекста из обратной связи

```python
# Псевдокод update_context_from_feedback()

1. Прочитать feedback_log.md (новые записи с прошлого обновления)
2. Если есть новые записи:
   a. Сформировать промпт для Qwen:
      "Проанализируй ошибки категоризации и обнови правила"
   b. Qwen генерирует:
      - Новые примеры для categories_context.md
      - Обновлённые правила
      - Новые заметки о частых ошибках
   c. Обновить categories_context.md
   d. Очистить feedback_log.md (или перенести в архив)
   e. Отправить уведомление:
      "🧠 Контекст категоризации обновлён: 3 новых правила"
```

### 7.4. Интеграция с FastAPI

```python
# lifespan.py
from contextlib import asynccontextmanager
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler()

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup
    scheduler.start()
    scheduler.add_job(generate_recurring_tasks, 'cron', hour=0, minute=5)
    scheduler.add_job(cleanup_missed_messages, 'cron', hour=3)
    # ... другие задачи
    
    # Запуск воркера AI очереди
    asyncio.create_task(ai_queue_worker())
    
    yield
    
    # Shutdown
    scheduler.shutdown()
```

## 8. AI категоризация

### 8.1. Модель

- **Модель:** Qwen (через transformers) или Dashscope API (облако)
- **Формат:** HuggingFace (локально) или API (облако)
- **Задача:** Классификация текста задачи
- **Рекомендация:** Начать с локального Qwen, при нехватке RAM перейти на Dashscope API

### 8.1.1. Контекстный файл для категоризации

Файл `config/categories_context.md` содержит:

```markdown

## Структура категорий
🏢 Работа
├── Финансы (маржа, аналитика, подсчёт денег по проектам)
├── Документы (реестры, инлайн-документация)
├── Проекты (зелёный марафон, майоли, ТЗ для клиентов)
├── Инфраструктура (серверы, деплой, VPN для работы)
└── Контент (статьи, выступления для работы)

🏠 Личное
├── Семья (Даня, подарки)
├── Здоровье (врач, массаж, баня)
├── Финансы (оплата света, личные финансы)

## Примеры задач и их категорий

| Задача | Категория | Подкатегория | Почему |
|--------|-----------|--------------|--------|
| алина ку3 | Работа | Финансы | Маржинальность по проекту |
| проверить инлайн | Работа | Документы | Инлайн — это документация |
| установить впн ЗМ | Работа | Проекты | ЗМ = зелёный марафон (рабочий проект) |
| ТЗ на сайд Дони | Личное | Свои сайты | Дони — садоводство, не клиент |
| ТЗ на свой сайт | Личное | Свои сайты | Свой сайт — не работа |
| завершить планербот | Личное | Пет-проекты | Пет-проект, не клиентская работа |
| транскрибатор на хостинге | Личное | Пет-проекты | Пет-проект |
| маржа майоли январь | Работа | Финансы | Майоли — рабочий проект |

## Правила категоризации

1. Если задача про деньги и это НЕ личный счёт → Работа/Финансы
2. Если проект для клиента → Работа/Проекты
3. Если проект для себя → Личное/Пет-проекты
4. Если сайт для себя или садоводства → Личное/Свои сайты
5. Если сайт для клиента → Работа/Проекты
6. ТЗ для клиентов → Работа/Проекты, для себя → Личное/Свои сайты
7. VPN/серверы для работы → Работа/Инфраструктура
```

## 9. OCR скриншотов

### 9.1. Технология

- **Библиотека:** Apple Vision Framework (macOS-native, бесплатно, быстро) или PaddleOCR (fallback)
- **Формат:** PNG/JPG
- **Язык:** русский + английский
- **Preprocessing:** Resize до макс. 2000px, конвертация в grayscale для ускорения

### 9.2. Формат извлечения

```
Вход: скриншот календаря
Выход: список событий

[
  {"time": "09:00", "title": "Григорьева пр...", "duration": "1 час"},
  {"time": "11:00-12:00", "title": "Планирование...", "duration": "1 час"},
  {"time": "12:00", "title": "Группхедский статус", "duration": null}
]
```

### 9.3. Обработка ошибок OCR

- Если OCR не распознал текст → сохранить скриншот в БД
- Создать задачу "Распознать вручную" с ссылкой на изображение
- Пользователь может вручную ввести события из скриншота

## 16. Риски и ограничения

| Риск | Митигация |
|------|-----------|
| Qwen требует много RAM (~4-8GB) | Использовать quantized версию (4-bit) или Dashscope API |
| OCR неточен на скриншотах | Ручная корректировка в UI, Apple Vision Framework |
| Ноут выключен → бот не работает | Синхронизация при включении |
| GigaAM интеграция сложная | Использовать существующий hf_patch.py |
| SQLite contention | WAL mode + busy_timeout + асинхронная очередь |
